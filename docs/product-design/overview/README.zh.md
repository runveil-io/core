# 项目概览

## 目的

本节定义 Veil 是什么、它要优化什么，以及它明确不打算成为什么。

## 适用场景

- 你第一次进入项目
- 你需要最短路径理解 Veil 的产品定位
- 你想在看架构或代码前先明确项目范围

## Veil 是什么

Veil 正在被构建成一个开源 AI 容量市场。它当前的运行时首先表现为一个 AI 推理路由系统：为调用方提供本地 OpenAI 兼容网关，并通过 Relay 把加密请求路由到可用的 Provider，让 AI 访问不必在每个客户端里都与某个上游账户一一绑定。

## 长期产品目标

Veil 正在朝着一个由 Agent 运营的 AI 容量市场演进。目标体验是：运维者只需要告诉 Claw 接入网络并开始出售可用推理容量，运行时就能自动完成发现、注册、定价、路由、见证记录和结算，而不需要大量人工介入。

这个长期目标不是后期附加想法，而是来自 [产品愿景](../../design-governance/product-vision.zh.md) 的延续。

## 核心价值

Veil 聚焦五件事：

- 与现有 AI 客户端兼容
- 通过本地网关和可路由供给实现无账户接入
- 分离控制平面可见性与执行平面可见性
- 复用 Provider 侧的可用供给
- 为预算、见证记录与对账建立证据边界
- 在不把运行时做成链上优先系统的前提下，把结算能力保留在主路径上

## 产品方向

- Consumer 通过一个本地接口访问可用模型供给
- Provider 把闲置 AI 配额或 API 容量转换成可路由执行能力
- Relay 负责撮合供给、执行路由规则并记录见证记录
- Claw 会从开发辅助工具演进成节点接入、运营和变现的自动化层
- 即使当前运行时仍以报价单位和离线证据为主，crypto 原生支付与结算方向也保留在产品主线上
- 开源构建循环保持协议演进过程可见、可审计

## 主要角色

- `Consumer`：运行本地网关并发送请求
- `Relay`：验签、路由、限流、记录见证记录
- `Provider`：解密、执行上游推理、返回结果
- `Bootstrap`：发布 Relay 可用性

## 非目标

- 不加限定的匿名承诺
- 在市场闭环成立之前先把运行时做成链上优先系统
- 通用计算执行平台
- 把绕过上游规则作为公开能力
